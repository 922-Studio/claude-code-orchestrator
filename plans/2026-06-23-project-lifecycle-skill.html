ADD playbook

pending

Author skills/project-lifecycle/add.md — the agentic playbook invoked as “build new project X like <ref>”. Phased, each phase self-loads context via pointers and gates on confirmation before side effects.

  • Phase 1 Define: name, archetype/reference project, type, deps, deployed?, domain, auth/DB/Celery — run project-lifecycle.sh preflight for collisions (name, port, redis-db, domain)
  • Phase 2 GitHub: gh repo create 922-Studio/<name>, scaffold from archetype files, seed CI from 922-Studio/workflows, push main+dev, set dev default
  • Phase 3 Local: clone to /Users/gregor/dev/922/<name>, .env, verify local run
  • Phase 4 Server: drive new-service-setup.md (db:create, redis db, compose, Traefik/Cloudflare, deploy.sh, first deploy) — each ssh lab step shown before execution
  • Phase 5 Monitoring: register in HomeCollector DEFAULT_MONITORED_SERVICES + HomeAPI versioning (mandatory for APIs)
  • Phase 6 Docs: registry.md row + type group + graph + deps; new projects/<name>.md from template; server.md; showcase.md (if showcase-worthy)
  • Phase 7 Verify: /health + /version reachable, monitoring green, CI green — report all URLs

03

REMOVE playbook (safety-gated)

pending

Author skills/project-lifecycle/remove.md — the inverse, ordered so teardown never leaves orphaned data or permanent false alerts. Presents an approval table (mirroring cleanup’s “type execute” gate) before any destructive action.

  • Order: (1) remove from HomeCollector monitoring → (2) remove from HomeAPI versioning → (3) stop/remove containers → (4) backup then drop DB + free redis db → (5) strip Traefik/Cloudflare/Prometheus → (6) orchestrator docs (registry, archive projects/<name>.md, server.md, showcase) → (7) archive plans referencing it
  • Uses project-lifecycle.sh audit <name> to enumerate every reference before touching anything
  • Never auto-deletes the GitHub repo or remote branches — repo disposition is an explicit manual confirmation
  • DB drop requires a captured backup path first; destructive steps behind the approval gate

Helper script

pending

Author scripts/project-lifecycle.sh — pure data-gatherer (no edits), mirroring orchestrator-audit.sh. The skill reasons; the script just surfaces state in one shot.

  • set -euo pipefail, repo-root-relative paths, no destructive operations
