Locked decisions
Light tint scopeTint the page background only — --hl-bg shifts to a cool blue-grey (hue ~220, very low chroma); --hl-bg-1 (card surface) stays pure white. Cards then "lift" off the tinted page, which is the look Gregor described. Hover/active surfaces and sidebar bg gain a hair more tint than --hl-bg so they remain distinguishable.
Tint hue choiceHue 220 (the same cool hue used by the dark-mode surfaces) at chroma ~0.006 — a desaturated blue-grey. Stays clear of teal accent (hue 195) so the accent still reads as the accent, not as "background overflow".
Ledger palette familyMove OFF semantic green/red. Use accent-soft tones for positive amounts (teal family, matches the app brand) and neutral muted tones for negative amounts (no warning hue — just --hl-fg at full weight against a neutral chip). Rationale: a ledger row is a stateful identity, not an alert; saturating it as success/error makes Finance feel like a stock-trading app. Accent-positive + neutral-negative reads as "owes the app vs. the app owes them" without screaming. Avatars get the same family: --hl-avatar--success → accent-soft chip, --hl-avatar--error → neutral chip with stronger border.
Pastel tokens fateRepurpose: --hl-success-pastel becomes --hl-ledger-pos (accent-derived). --hl-error-pastel becomes --hl-ledger-neg (neutral-fg-derived). Old token names removed; ledger is the only consumer so the rename is safe. Saturated --hl-success / --hl-error stay untouched — banners, pills, table pos/neg cells keep their semantics.
Panel equalisationRecommended: grid container = stretch + child flex column. Add align-items: stretch (default but make it explicit and add equal-row-height rules) to .hl-grid; change .hl-card and .hl-stat to display: flex; flex-direction: column; mark .hl-card__desc with flex: 1 so footers align at the bottom of each card. For .hl-section panels that sit side-by-side (GitHub Actions runner-grid vs. runs-table), wrap them in a parent .hl-grid--2 (new utility if missing — check showcase first) with stretch; the runs-table panel gets display: flex; flex-direction: column and its inner content takes the remaining height. No min-height hacks.

pending

Edit the .light override block in src/styles/design-system.css (lines ~819–857). Shift surface tokens from warm hue 75 to cool hue 220 with very low chroma. Keep --hl-bg-1 pure white so cards lift off the page.

03

Panel-size equalisation — DS CSS

pending

Equalise card / stat / section heights inside grids. All edits in design-system.css.

  • .hl-grid (L788): add align-items: stretch explicitly and verify grid-auto-rows: 1fr is NOT applied (stretch on align is enough for equal-height children in the same row)
  • .hl-card (L377): add display: flex; flex-direction: column; so inner blocks can grow
  • .hl-card__desc (L408): add flex: 1; so cards with longer descriptions push the footer down and cards with shorter descriptions still align footers at the bottom row
  • .hl-stat (L428): already display: flex; flex-direction: column; — add justify-content: flex-start; + height: 100%; so stats fill their grid cell when neighbours are taller
  • .hl-section (L345): currently just margin-bottom — when used as a grid child (e.g. GitHub Actions runner-grid vs. runs-table side-by-side), it must also fill height. Add .hl-section defaults: display: flex; flex-direction: column; and make the section's primary content block the flex-grow target. Verify this doesn't break pages where .hl-section is a top-level page wrapper (it currently is on most pages — flex-column with no growers is a no-op visually but verify margin behaviour)
  • Edge case — very-short content: cards with 1-line descriptions inside a row of 3-line-description neighbours should still look intentional. Verify that flex: 1 on .hl-card__desc doesn't create awkward whitespace gaps; if it does, prefer align-self: stretch on the desc + a minimum content block instead

If the GitHub Actions side-by-side panels (runner-grid .hl-section + runs-table .hl-card) need a 2-col grid utility and .hl-grid--2 is missing from pages-design-system.css showcase, add it here as the obvious sibling of --3 / --4 / --5 (L789–791).
