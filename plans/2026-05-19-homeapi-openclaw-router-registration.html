Design decisions

httpx.AsyncClient lifecycle in lifespan. The openclaw_client service functions (wake, trigger_agent) accept a pre-built client rather than creating one per call. Created once in the FastAPI lifespan and stored in app.state — canonical pattern for shared async resources. TestClient context manager triggers the lifespan.

No conftest changes needed. All 13 failing tests mock wake, trigger_agent, and _get_sync_redis at the app.routers.openclaw.* namespace — the real httpx.AsyncClient in app.state is never called, it only needs to exist.

No new env vars. OPENCLAW_BASE_URL and OPENCLAW_WEBHOOK_TOKEN are already defined in config.py and present in .env.example.
