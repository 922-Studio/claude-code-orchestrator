You assist with logging and querying personal tracking data via the homeapi MCP tools.

You are the sensei agent for Gregor's personal Discord server.

- No external tools — pure reasoning and explanation
- Calibrate depth to a senior full-stack developer; skip basics
- Keep responses concise; offer to elaborate on request

- Follow the user's language (German or English)

discord/services/claude_runner.py — design

ResponsibilityImplementation
Accept message + agent configasync def run(channel_id, message, agent_config) → str
Load rolling historyRedis key claude_history:{channel_id} — LRANGE 0 9 (5 pairs)
Build promptsystem persona + history text + current message, single string
Invoke claudeclaude -p "<prompt>" [--mcp-config <path>] --no-cwd via asyncio subprocess
Timeout60s hard kill; return "[timeout — no response]" to Discord
Update historyLPUSH user/assistant pair, LTRIM to 10 items
Error handlingNon-zero exit → log stderr, return generic error (never expose stderr)

No persistent subprocess — each message is a fresh claude -p call; history is injected as text context. Simple, no zombie processes, sufficient for a 5-message window.

discord/cogs/claude_ai.py — design

  • Auto-discovered by existing cog loader — no changes to bot.py
  • on_message listener filtered to channels in CLAUDE_AGENTS
  • Ignore messages starting with ; (preserves bot commands)
  • Ignore messages from bots (including self)
  • Show typing indicator (async with channel.typing()) while awaiting runner
  • Post response; on error post short failure notice (no internal detail)
