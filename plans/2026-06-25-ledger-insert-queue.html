Locked design (confirmed with Gregor 2026-06-25)
  • Insert sheet = the depts sheet's FIRST tab. Columns: Person | Amount | Note (manual). No date column → transaction_date = processing date.
  • Queue semantics: a 5-min Celery-beat job reads every row, resolves/auto-creates the Person (in org), coerces+validates Amount (Decimal), persists idempotently (transaction_date = today), and DELETES the row once confirmed. Persist-then-delete ordering; malformed rows are NOT deleted — they stay with a per-row error note. Resilience: if the system is down, rows wait in the queue and process when it returns.
  • Person tabs are read-only mirrors — sole input path is the insert sheet. Drop per-person manual import; export still rewrites person tabs (timeline order).
  • Feature flag (org-level) toggles the whole intake on/off, set from a HomeUI setting. Beat job is a no-op when disabled. Interval fixed at 5 min.
