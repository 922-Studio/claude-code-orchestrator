GitHub Actions artifact storage hit quota. Stop uploading any test artifacts to GitHub (all test diagnostics, including Playwright traces, live on the Allure server), and store only runner execution detail in a new dedicated MinIO bucket with 30-day retention, reached directly over Tailscale.

Decisions locked
  • No test artifacts on GitHub. Remove every actions/upload-artifact step, including Playwright traces — those attach to Allure instead.
  • MinIO bucket stores runner execution detail only (job logs + run metadata), 30-day lifecycle expiry.
  • Network path: direct over Tailscale — runners push to MinIO on antares's tailnet IP, bypassing Cloudflare's ~100MB body limit.
Open question to resolve in Step 01
  • What is "runner execution detail" exactly, and how to capture it? Two approaches:
    • (A) Inline final step — a last job step assembles a JSON manifest (repo, run_id, run_attempt, job, runner host, commit, branch, event, conclusion, per-step durations, test summary) and pushes it to MinIO. Cheap, but cannot include the job's own complete live log (the job is still running).
    • (B) workflow_run-triggered archival (recommended) — a separate reusable workflow fires when a CI run completes, calls gh run view <id> --log + gh api for timing/metadata, zips it, and pushes the complete bundle to MinIO. Gets the full log; one extra short job per run.
