Backend: paid status + notes + mark-paid endpoints

pending

Extend the Invoice model with payment-tracking fields and add explicit mark-paid / mark-unpaid / update-notes endpoints.

  • Migration u001_add_invoice_paid_and_notes.py chained off t002: add columns paid_at TIMESTAMP NULL, paid_amount NUMERIC(12,2) NULL (nullable; we don't enforce partial payment yet), user_notes TEXT NULL (the user-editable note distinct from the immutable notes snapshot text). Extend the invoice_status enum with paid (use ALTER TYPE invoice_status ADD VALUE 'paid'; the migration must commit before any code references the new value — wrap in op.execute; downgrade can leave the value, Postgres doesn't easily remove enum members).
  • Update app/models/invoice.py + app/schemas/invoice.py (add the new fields to Invoice response; InvoiceListItem too).
  • New endpoints in app/routers/finance/invoices.py: POST /{id}/mark-paid body {paid_at?: date, paid_amount?: Decimal} → sets status='paid', paid_at = body.paid_at or today, paid_amount = body.paid_amount or invoice.total_amount. 200 with Invoice; POST /{id}/mark-unpaid → reverts to status='issued', clears paid_at/paid_amount; PATCH /{id} body {user_notes?: str} → updates the editable note (status-agnostic).
  • All endpoints enforce org_id + 404 on missing. Cancelled invoices may not be marked paid (422 with clear message). Paid invoices may not be cancelled (422). Add CRUD helpers in app/crud/invoice.py.
  • Unit tests for the new CRUD; integration tests for the new endpoints (happy path + 422 transitions + 404 + cross-org isolation).
  • Alembic head goes t002 → u001. start.sh applies on container restart.
